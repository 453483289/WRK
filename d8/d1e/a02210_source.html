23 //
24 // The Multi-Sector Header and Update Sequence Array provide detection of
25 // incomplete multi-sector transfers for devices which either have a
26 // physical sector size equal to the Sequence Number Stride or greater, or
27 // which do not transfer sectors out of order. If a device exists which has
28 // a sector size smaller than the Sequence Number Stride *and* it sometimes
29 // transfers sectors out of order, then the Update Sequence Array will not
30 // provide absolute detection of incomplete transfers. The Sequence Number
31 // Stride is set to a small enough number to provide absolute protection for
32 // all known hard disks. It is not set any smaller, in order to avoid
33 // excessive run time and space overhead.
34 //
35 // The Multi-Sector Header contains space for a four-byte signature for the
36 // convenience of its user. It then provides the offset to and length of the
37 // the Update Sequence Array. The Update Sequence Array consists of an array
38 // of n saved USHORTs, where n is the size of the structure being protected
39 // divided by the sequence number stride. (The size of structure being
40 // protected must be a nonzero power of 2 times the Sequence Number Stride,
41 // and less than or equal to the physical page size of the machine.) The
42 // first word of the Update Sequence Array contains the Update Sequence Number,
43 // which is a cyclical counter (however 0 is not used) of the number of times
44 // the containing structure has been written to disk. Following the Update
45 // Sequence Number are the n saved USHORTs which were overwritten by the
46 // Update Sequence Number the last time the containing structure was
47 // written to disk.
48 //
49 // In detail, just prior to each time the protected structure is written to
50 // disk, the last word in each Sequence Number Stride is saved to its
51 // respective position in the Sequence Number Array, and then it is overwritten
52 // with the next Update Sequence Number. Just after this write, or whenever
53 // reading the structure, the saved word from the Sequence Number Array is
54 // restored to its actual position in the structure. Before restoring the
55 // saved words on reads, all of the sequence numbers at the end of each
56 // stride are compared with the actual sequence number at the start of the
57 // array. If any of these compares come up not equal, then a failed
58 // multi-sector transfer has been detected.
59 //
60 // The size of the array is determined by the size of the containing structure.
61 // As a C detail, the array is declared here with a size of 1, since its
62 // actual size can only be determined at runtime.
63 //
64 // The Update Sequence Array should be included at the end of the header of
65 // the structure it is protecting, since it is variable size. Its user must
66 // ensure that the correct size is reserved for it, namely:
67 //
68 // (sizeof-structure / SEQUENCE_NUMBER_STRIDE + 1) * sizeof(USHORT)
69 //
70 
71 #define SEQUENCE_NUMBER_STRIDE (512)
