35 /*++
36 
37 Routine Description:
38 
39  This procedure is used by the server process to wait for a message from a
40  client process
41 
42  A client and server process can receive messages using the
43  NtReplyWaitReceivePort service:
44 
45  If the ReplyMessage parameter is specified, then the reply will be sent
46  using NtReplyPort.
47 
48  If the PortHandle parameter specifies a connection port, then the receive
49  will return whenever a message is sent to a server communication port that
50  does not have its own receive queue and the message is therefore queued to
51  the receive queue of the connection port.
52 
53  If the PortHandle parameter specifies a server communication port that
54  does not have a receive queue, then behaves as if the associated
55  connection port handle was specified. Otherwise the receive will return
56  whenever message is placed in the receive queue associated with the
57  server communication port.
58 
59  The received message will be returned in the variable specified by the
60  ReceiveMessage parameter. If the MapInfoOffset field of the reply message
61  is non-zero, then the PORT_MAP_INFORMATION structure it points to will be
62  processed and the relevant pages will be mapped into the caller's address
63  space. The service returns an error if there is not enough room in the
64  caller's address space to accommodate the mappings.
65 
66 Arguments:
67 
68  PortHandle - Specifies the handle of the connection or communication port
69  to do the receive from.
70 
71  PortContext - Specifies an optional pointer to a variable that is to
72  receive the context value associated with the communication port that
73  the message is being received from. This context variable was
74  specified on the call to the NtAcceptConnectPort service.
75 
76  ReplyMessage - This optional parameter specifies the address of a reply
77  message to be sent. The ClientId and MessageId fields determine which
78  thread will get the reply. See description of NtReplyPort for how the
79  reply is sent. The reply is sent before blocking for the receive.
80 
81  ReceiveMessage - Specifies the address of a variable to receive the
82  message.
83 
84 Return Value:
85 
86  Status code that indicates whether or not the operation was successful.
87 
88 --*/
89 
90 {
91  PAGED_CODE();
92 
93  return NtReplyWaitReceivePortEx (PortHandle,
94  PortContext,
95  ReplyMessage,
96  ReceiveMessage,
97  NULL);
98 }
