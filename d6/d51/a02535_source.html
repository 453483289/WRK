614 /*++
615 
616 Routine Description:
617 
618  This routine is called by the transaction filesystem to determine if
619  the given transaction is referencing a file which has user writable sections
620  or user writable views into it. If so, the transaction must be aborted
621  as it cannot be guaranteed atomicity.
622 
623  The transaction filesystem is responsible for checking and intercepting
624  file object creates that specify write access prior to using this
625  interface. Specifically, prior to starting a transaction, the transaction
626  filesystem must ensure that there are no writable file objects that
627  currently exist for the given file in the transaction. While the
628  transaction is ongoing, requests to create file objects with write access
629  for the transaction files must be refused.
630 
631  This Mm routine exists to catch the case where the user has closed the
632  file handles and the section handles, but still has open writable views.
633 
634  For this reason, no locks are needed to read the value below.
635 
636 Arguments:
637 
638  SectionPointer - Supplies a pointer to the section object pointers
639  from the file object.
640 
641 Return Value:
642 
643  Number of user writable references.
644 
645 Environment:
646 
647  Kernel mode, APC_LEVEL or below, no mutexes held.
648 
649 --*/
650 
651 {
652  KIRQL OldIrql;
653  ULONG WritableUserReferences;
654  PCONTROL_AREA ControlArea;
655 
656  WritableUserReferences = 0;
657 
658  LOCK_PFN (OldIrql);
659 
660  ControlArea = (PCONTROL_AREA)(SectionPointer->DataSectionObject);
661 
662  if (ControlArea != NULL) {
663  WritableUserReferences = ControlArea->WritableUserReferences;
664  }
665 
666  UNLOCK_PFN (OldIrql);
667 
668  return WritableUserReferences;
669 }
