64 /*++
65 
66 Routine Description:
67 
68  A client and server process can send a reply to a previous request
69  message with the NtReplyPort service:
70 
71  The Type field of the message is set to LPC_REPLY by the service. If the
72  MapInfoOffset field of the reply message is non-zero, then the
73  PORT_MAP_INFORMATION structure it points to will be processed and the
74  relevant pages in the caller's address space will be unmapped.
75 
76  The ClientId and MessageId fields of the ReplyMessage structure are used
77  to identify the thread waiting for this reply. If the target thread is
78  in fact waiting for this reply message, then the reply message is copied
79  into the thread's message buffer and the thread's wait is satisfied.
80 
81  If the thread is not waiting for a reply or is waiting for a reply to
82  some other MessageId, then the message is placed in the message queue of
83  the port that is connected to the communication port specified by the
84  PortHandle parameter and the Type field of the message is set to
85  LPC_LOST_REPLY.
86 
87 Arguments:
88 
89  PortHandle - Specifies the handle of the communication port that the
90  original message was received from.
91 
92  ReplyMessage - Specifies a pointer to the reply message to be sent.
93  The ClientId and MessageId fields determine which thread will
94  get the reply.
95

97
