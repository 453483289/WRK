13 Abstract:
14 
15  Security implementation for WMI objects
16 
17  WMI security is guid based, that is each guid can be assigned a security
18  descriptor. There is also a default security descriptor that applies
19  to any guid that does not have its own specific security descriptor.
20  Security is enforced by relying upon the object manager. We define the
21  WmiGuid object type and require all WMI requests to have a handle to the
22  WmiGuid object. In this way the guid is opened with a specific ACCESS_MASK
23  and if the caller is permitted those rights (as specified in the specific
24  or default security descriptor) then a handle is returned. When the
25  caller wants to do an operation he must pass the handle and before the
26  operation is performed we check that the handle has the allowed access.
27 
28  Guid security descriptors are serialized as REG_BINARY values under the
29  registry key HKLM\CurrentControlSet\Control\Wmi\Security. If no specific
30  or default security descriptor for a guid exists then the all access
31  is available for anyone. For this reason this registry key must be
32  protected.
33 
34  WMI implements its own security method for the WmiGuid object type to
35  allow it to intercept any changes to an objects security descriptor. By
36  doing this we allow the standard security apis
37  (Get/SetKernelObjectSecurity) to query and set the WMI security
38  descriptors.
39 
40  A guid security descriptor contains the following specific rights:
41 
42  WMIGUID_QUERY 0x0001
43  WMIGUID_SET 0x0002
44  WMIGUID_NOTIFICATION 0x0004
45  WMIGUID_READ_DESCRIPTION 0x0008
46  WMIGUID_EXECUTE 0x0010
47  TRACELOG_CREATE_REALTIME 0x0020
48  TRACELOG_CREATE_ONDISK 0x0040
49  TRACELOG_GUID_ENABLE 0x0080
50  TRACELOG_ACCESS_KERNEL_LOGGER 0x0100
51 --*/

171 WmipSecurityMethod (
172  IN PVOID Object,
173  IN SECURITY_OPERATION_CODE OperationCode,
174  IN PSECURITY_INFORMATION SecurityInformation,
175  IN OUT PSECURITY_DESCRIPTOR SecurityDescriptor,
176  IN OUT PULONG CapturedLength,
177  IN OUT PSECURITY_DESCRIPTOR *ObjectsSecurityDescriptor,
178  IN POOL_TYPE PoolType,
179  IN PGENERIC_MAPPING GenericMapping
180  )
181 
182 /*++
183 
184 Routine Description:
185 
186  This is the WMI security method for objects. It is responsible
187  for either retrieving, setting, and deleting the security descriptor of
188  an object. It is not used to assign the original security descriptor
189  to an object (use SeAssignSecurity for that purpose).
190 
191 
192  IT IS ASSUMED THAT THE OBJECT MANAGER HAS ALREADY DONE THE ACCESS
193  VALIDATIONS NECESSARY TO ALLOW THE REQUESTED OPERATIONS TO BE PERFORMED.
194 
195  This code stolen directly from SeDefaultObjectMethod in
196  \nt\private\ntos\se\semethod.c. It does not do anything special except
197  serialize any SD that is being set for an object.
198 
199 Arguments:
200 
201  Object - Supplies a pointer to the object being used.
202 
203  OperationCode - Indicates if the operation is for setting, querying, or
204  deleting the object's security descriptor.
205 
206  SecurityInformation - Indicates which security information is being
207  queried or set. This argument is ignored for the delete operation.
208 
209  SecurityDescriptor - The meaning of this parameter depends on the
210  OperationCode:
211 
212  QuerySecurityDescriptor - For the query operation this supplies the
213  buffer to copy the descriptor into. The security descriptor is
214  assumed to have been probed up to the size passed in in Length.
215  Since it still points into user space, it must always be
216  accessed in a try clause in case it should suddenly disappear.
217 
218  SetSecurityDescriptor - For a set operation this supplies the
219  security descriptor to copy into the object. The security
220  descriptor must be captured before this routine is called.
221 
222  DeleteSecurityDescriptor - It is ignored when deleting a security
223  descriptor.
224 
225  AssignSecurityDescriptor - For assign operations this is the
226  security descriptor that will be assigned to the object.
227  It is assumed to be in kernel space, and is therefore not
228  probed or captured.
229 
230  CapturedLength - For the query operation this specifies the length, in
231  bytes, of the security descriptor buffer, and upon return contains
232  the number of bytes needed to store the descriptor. If the length
233  needed is greater than the length supplied the operation will fail.
234  It is ignored in the set and delete operation.
235 
236  This parameter is assumed to be captured and probed as appropriate.
237 
238  ObjectsSecurityDescriptor - For the Set operation this supplies the address
239  of a pointer to the object's current security descriptor. This routine
240  will either modify the security descriptor in place or allocate a new
241  security descriptor and use this variable to indicate its new location.
242  For the query operation it simply supplies the security descriptor
243  being queried. The caller is responsible for freeing the old security
244  descriptor.
245 
246  PoolType - For the set operation this specifies the pool type to use if
247  a new security descriptor needs to be allocated. It is ignored
248  in the query and delete operation.
249 
250  the mapping of generic to specific/standard access types for the object
251  being accessed. This mapping structure is expected to be safe to
252  access (i.e., captured if necessary) prior to be passed to this routine.
253 
254 Return Value:
255 
256  NTSTATUS - STATUS_SUCCESS if the operation is successful and an
257  appropriate error status otherwise.
258 
259 --*/
260 
261 {
262  NTSTATUS Status;
263 
264  PAGED_CODE();
265 
266  //
267  // If the object's security descriptor is null, then object is not
268  // one that has security information associated with it. Return
269  // an error.
270  //
271 
272  //
273  // Make sure the common parts of our input are proper
274  //
275 
276  ASSERT( (OperationCode == SetSecurityDescriptor) ||
277  (OperationCode == QuerySecurityDescriptor) ||
278  (OperationCode == AssignSecurityDescriptor) ||
279  (OperationCode == DeleteSecurityDescriptor) );
280 
281  //
282  // This routine simply cases off of the operation code to decide
283  // which support routine to call
284  //
285 
286  switch (OperationCode) {
287 
288  case SetSecurityDescriptor:
289  {
290  UNICODE_STRING GuidName;
291  WCHAR GuidBuffer[38];
292  LPGUID Guid;
293  SECURITY_INFORMATION LocalSecInfo;
294  PSECURITY_DESCRIPTOR SecurityDescriptorCopy;
295  ULONG SecurityDescriptorLength;
296  NTSTATUS Status2;
297 
298  ASSERT( (PoolType == PagedPool) || (PoolType == NonPagedPool) );
299 
300  Status = ObSetSecurityDescriptorInfo( Object,
301  SecurityInformation,
302  SecurityDescriptor,
303  ObjectsSecurityDescriptor,
304  PoolType,
305  GenericMapping
306  );
307 
308  if (NT_SUCCESS(Status))
309  {
310  //
311  // Serialize the guid's new security descriptor in
312  // the registry. But first we need to get a copy of
313  // it.
314 
315  SecurityDescriptorLength = 1024;
316  do
317  {
318  SecurityDescriptorCopy = ExAllocatePoolWithTag(
319  PoolType,
320  SecurityDescriptorLength,
321  WMIPOOLTAG);
322 
323  if (SecurityDescriptorCopy == NULL)
324  {
325  Status2 = STATUS_INSUFFICIENT_RESOURCES;
326  break;
327  }
328  LocalSecInfo = 0xffffffff;
329  Status2 = ObQuerySecurityDescriptorInfo( Object,
330  &LocalSecInfo,
331  SecurityDescriptorCopy,
332  &SecurityDescriptorLength,
333  ObjectsSecurityDescriptor);
334 
335 
336  if (Status2 == STATUS_BUFFER_TOO_SMALL)
337  {
338  ExFreePool(SecurityDescriptorCopy);
339  } else {
340  break;
341  }
342 
343  } while (TRUE);
344 
345 
346  if (NT_SUCCESS(Status2))
347  {
348  Guid = &((PWMIGUIDOBJECT)Object)->Guid;
349  StringCbPrintf(GuidBuffer,
350  sizeof(GuidBuffer),
351  L"%08x-%04x-%04x-%02x%02x-%02x%02x%02x%02x%02x%02x",
352  Guid->Data1, Guid->Data2,
353  Guid->Data3,
354  Guid->Data4[0], Guid->Data4[1],
355  Guid->Data4[2], Guid->Data4[3],
356  Guid->Data4[4], Guid->Data4[5],
357  Guid->Data4[6], Guid->Data4[7]);
358 
359  RtlInitUnicodeString(&GuidName, GuidBuffer);
360 
361  WmipSaveGuidSecurityDescriptor(&GuidName,
362  SecurityDescriptorCopy);
363  }
364 
365  if (SecurityDescriptorCopy != NULL)
366  {
367  ExFreePool(SecurityDescriptorCopy);
368  }
369 
370  }
371 
372  return(Status);
373  }
374 
375 
376 
377  case QuerySecurityDescriptor:
378  {
379 
380  //
381  // check the rest of our input and call the default query security
382  // method
383  //
384 
385  ASSERT( CapturedLength != NULL );
386 
387 
388  return ObQuerySecurityDescriptorInfo( Object,
389  SecurityInformation,
390  SecurityDescriptor,
391  CapturedLength,
392  ObjectsSecurityDescriptor );
393  }
394 
395  case DeleteSecurityDescriptor:
396  {
397 
398  //
399  // call the default delete security method
400  //
401 
402  Status = ObDeassignSecurity(ObjectsSecurityDescriptor);
403  return(Status);
404  }
405 
406  case AssignSecurityDescriptor:
407 
408  ObAssignObjectSecurityDescriptor( Object,
409  SecurityDescriptor,
410  PoolType );
411  return( STATUS_SUCCESS );
412 
413  default:
414 
415  //
416  // Bugcheck on any other operation code, We won't get here if
417  // the earlier asserts are still checked.
418  //
419 
420  KeBugCheckEx( SECURITY_SYSTEM, 1, (ULONG_PTR) STATUS_INVALID_PARAMETER, 0, 0 );
421  }

473  }
474

623

625  {

627  }
628 
629  return(Status);

746 NTSTATUS WmipGetGuidSecurityDescriptor(
747  IN PUNICODE_STRING GuidName,
748  IN PSECURITY_DESCRIPTOR *SecurityDescriptor,
749  IN PSECURITY_DESCRIPTOR UserDefaultSecurity
750  )
751 /*++
752 
753 Routine Description:
754 
755  This routine will retrieve the security descriptor associated with a
756  guid. First it looks for a security descriptor specifically for the
757  guid and if not found then looks for the default security descriptor.
758 
759  Security descriptors are maintained as REG_BINARY values named by the guid
760  in the registry under
761  HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\Wmi\Security
762 
763 Arguments:
764 
765  GuidName is a pointer to a unicode string that represents the guid
766 
767  *SecurityDescriptor returns the security descriptor for the guid. It
768  must be freed back to pool unless it is the same value as that in
769  WmipDefaultAccessSd which must NOT be freed.
770 
771 Return Value:
772 
773  NT Status code
774 
775 --*/
776 {
777  RTL_QUERY_REGISTRY_TABLE QueryRegistryTable[3];
778  NTSTATUS Status;
779  PSECURITY_DESCRIPTOR GuidSecurityDescriptor = NULL;
780  PSECURITY_DESCRIPTOR DefaultSecurityDescriptor = NULL;
781 
782  PAGED_CODE();
783 
784  RtlZeroMemory(QueryRegistryTable, sizeof(QueryRegistryTable));
785 
786  QueryRegistryTable[0].QueryRoutine = WmipSDRegistryQueryRoutine;
787  QueryRegistryTable[0].EntryContext = &GuidSecurityDescriptor;
788  QueryRegistryTable[0].Name = GuidName->Buffer;
789  QueryRegistryTable[0].DefaultType = REG_BINARY;
790 
791  QueryRegistryTable[1].QueryRoutine = WmipSDRegistryQueryRoutine;
792  QueryRegistryTable[1].Flags = 0;
793  QueryRegistryTable[1].EntryContext = &DefaultSecurityDescriptor;
794  QueryRegistryTable[1].Name = DefaultSecurityGuidName;
795  QueryRegistryTable[1].DefaultType = REG_BINARY;
796 
797  Status = RtlQueryRegistryValues(RTL_REGISTRY_CONTROL,
798  L"WMI\\Security",
799  QueryRegistryTable,
800  NULL,
801  NULL);
802 
803  *SecurityDescriptor = NULL;
804  if (NT_SUCCESS(Status))
805  {
806  //
807  // If there is a guid specific SD then choose that and free any

1353 //
