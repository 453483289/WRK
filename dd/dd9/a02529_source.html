9 Module Name:
10 
11  physical.c
12 
13 Abstract:
14 
15  This module contains the routines to manipulate physical memory from
16  user space.
17 
18  There are restrictions on how user controlled physical memory can be used.
19  Realize that all this memory is nonpaged and hence applications should
20  allocate this with care as it represents a very real system resource.
21 
22  Virtual memory which maps user controlled physical memory pages must be :
23 
24  1. Private memory only (ie: cannot be shared between processes).
25 
26  2. The same physical page cannot be mapped at 2 different virtual
27  addresses.
28 
29  3. Callers must have LOCK_VM privilege to create these VADs.
30 
31  4. Device drivers cannot call MmSecureVirtualMemory on it - this means
32  that applications should not expect to use this memory for win32k.sys
33  calls.
34 
35  5. NtProtectVirtualMemory only allows read-write protection on this
36  memory. No other protection (no access, guard pages, readonly, etc)
37  are allowed.
38 
39  6. NtFreeVirtualMemory allows only MEM_RELEASE and NOT MEM_DECOMMIT on
40  these VADs. Even MEM_RELEASE is only allowed on entire VAD ranges -
41  that is, splitting of these VADs is not allowed.
42 
43  7. fork() style child processes don't inherit physical VADs.
44 
45  8. The physical pages in these VADs are not subject to job limits.
46 
47 --*/
