58 /*++
59 
60 Routine Description:
61 
62  This routine changes the protection on a region of committed pages
63  within the virtual address space of the subject process. Setting
64  the protection on a range of pages causes the old protection to be
65  replaced by the specified protection value.
66 
67  Note if a virtual address is locked in the working set and the
68  protection is changed to no access, the page is removed from the
69  working set since valid pages can't be no access.
70 
71 Arguments:
72 
73  ProcessHandle - An open handle to a process object.
74 
75  BaseAddress - The base address of the region of pages
76  whose protection is to be changed. This value is
77  rounded down to the next host page address
78  boundary.
79 
80  RegionSize - A pointer to a variable that will receive
81  the actual size in bytes of the protected region
82  of pages. The initial value of this argument is
83  rounded up to the next host page size boundary.
84 
85  NewProtectWin32 - The new protection desired for the specified region of pages.
86 
87  Protect Values
88 
89  PAGE_NOACCESS - No access to the specified region
90  of pages is allowed. An attempt to read,
91  write, or execute the specified region
92  results in an access violation.
93 
94  PAGE_EXECUTE - Execute access to the specified
95  region of pages is allowed. An attempt to
96  read or write the specified region results in
97  an access violation.
98 
99  PAGE_READONLY - Read only and execute access to the
100  specified region of pages is allowed. An
101  attempt to write the specified region results
102  in an access violation.
103 
104  PAGE_READWRITE - Read, write, and execute access to
105  the specified region of pages is allowed. If
106  write access to the underlying section is
107  allowed, then a single copy of the pages are
108  shared. Otherwise the pages are shared read
109  only/copy on write.
110 
111  PAGE_GUARD - Read, write, and execute access to the
112  specified region of pages is allowed,
113  however, access to the region causes a "guard
114  region entered" condition to be raised in the
115  subject process. If write access to the
116  underlying section is allowed, then a single
117  copy of the pages are shared. Otherwise the
118  pages are shared read only/copy on write.
119 
120  PAGE_NOCACHE - The page should be treated as uncached.
121  This is only valid for non-shared pages.
122 
123  PAGE_WRITECOMBINE - The page should be treated as write-combined.
124  This is only valid for non-shared pages.
125 
126  OldProtect - A pointer to a variable that will receive
127  the old protection of the first page within the
128  specified region of pages.
129 
130 Return Value:
131 
132  NTSTATUS.
133 
134 Environment:
135 
136  Kernel mode.
137 
138 --*/
