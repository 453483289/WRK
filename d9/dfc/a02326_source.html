69 
70 Routine Description:
71 
72  This service operates on a directory file or OLE container specified by the
73  FileHandle parameter. The service returns information about files in the
74  directory or embeddings and streams in the container specified by the file
75  handle. The ReturnSingleEntry parameter specifies that only a single entry
76  should be returned rather than filling the buffer. The actual number of
77  files whose information is returned, is the smallest of the following:
78 
79  o One entry, if the ReturnSingleEntry parameter is TRUE.
80 
81  o The number of entries whose information fits into the specified
82  buffer.
83 
84  o The number of entries that exist.
85 
86  o One entry if the optional FileName parameter is specified.
87 
88  If the optional FileName parameter is specified, then the only information
89  that is returned is for that single entries, if it exists. Note that the
90  file name may not specify any wildcard characters according to the naming
91  conventions of the target file system. The ReturnSingleEntry parameter is
92  simply ignored.
93 
94  The information that is obtained about the entries in the directory or OLE
95  container is based on the FileInformationClass parameter. Legal values are
96  hard coded based on the MinorFunction.
97 
98 Arguments:
99 
100  FileHandle - Supplies a handle to the directory file or OLE container for
101  which information should be returned.
102 
103  Event - Supplies an optional event to be set to the Signaled state when
104  the query is complete.
105 
106  ApcRoutine - Supplies an optional APC routine to be executed when the
107  query is complete.
108 
109  ApcContext - Supplies a context parameter to be passed to the ApcRoutine,
110  if an ApcRoutine was specified.
111 
112  IoStatusBlock - Address of the caller's I/O status block.
113 
114  FileInformation - Supplies a buffer to receive the requested information
115  returned about the contents of the directory.
116 
117  Length - Supplies the length, in bytes, of the FileInformation buffer.
118 
119  FileInformationClass - Specfies the type of information that is to be
120  returned about the files in the specified directory or OLE container.
121 
122  ReturnSingleEntry - Supplies a BOOLEAN value that, if TRUE, indicates that
123  only a single entry should be returned.
124 
125  FileName - Optionally supplies a file name within the specified directory
126  or OLE container.
127 
128  RestartScan - Supplies a BOOLEAN value that, if TRUE, indicates that the
129  scan should be restarted from the beginning. This parameter must be
130  set to TRUE by the caller the first time the service is invoked.
131 
132  MinorFunction - IRP_MN_QUERY_DIRECTORY or IRP_MN_QUERY_OLE_DIRECTORY
133 
134  SynchronousIo - pointer to returned BOOLEAN; TRUE if synchronous I/O
135 
136  DeviceObject - pointer to returned pointer to device object
137 
138  Irp - pointer to returned pointer to device object
139 
140  FileObject - pointer to returned pointer to file object
141 
142  RequestorMode - pointer to returned requestor mode

649  //

662  // clean everything up, and return an appropriate error status code.
663  //

665  IopExceptionCleanup( fileObject,
666  irp,
667  eventObject,
668  (PKEVENT) NULL );
669 
670  if (auxiliaryBuffer != NULL) {
671  ExFreePool( auxiliaryBuffer );
672  }
673 
674  return GetExceptionCode();
675 
676  }
677 
678  } else {
679 
680  //
681  // Pass the address of the user's buffer so the driver has access to
682  // it. It is now the driver's responsibility to do everything.
683  //
684 
685  irp->UserBuffer = FileInformation;
686 
687  }
688 
689  //
690  // Copy the caller's parameters to the service-specific portion of the
691  // IRP.
692  //
693 
694  irpSp->Parameters.QueryDirectory.Length = Length;
695  irpSp->Parameters.QueryDirectory.FileInformationClass = FileInformationClass;
696  irpSp->Parameters.QueryDirectory.FileIndex = 0;
697  irpSp->Parameters.QueryDirectory.FileName = (PUNICODE_STRING) auxiliaryBuffer;
698  irpSp->Flags = 0;
699  if (RestartScan) {
700  irpSp->Flags = SL_RESTART_SCAN;
701  }
702  if (ReturnSingleEntry) {
703  irpSp->Flags |= SL_RETURN_SINGLE_ENTRY;
704  }
705 
706  irp->Flags |= IRP_DEFER_IO_COMPLETION;
707 
708  //
709  // Return with everything set up for the caller to complete the I/O.
710  //
711 
712  return STATUS_SUCCESS;
713 }
