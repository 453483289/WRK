37 ;
38 ; Routine Description:
39 ;
40 ; This routine is called at system startup to perform early initialization
41 ; and to initialize the kernel debugger. This allows breaking into the
42 ; kernel debugger very early during system startup. After kernel debugger
43 ; initialization, kernel initialization is performed. On return from kernel
44 ; initialization the idle loop is entered. The idle loop begins execution
45 ; and immediately finds the system startup (phase 1) thread ready to run.
46 ; Phase 1 initialization is performed and all other processors are started.
47 ; As each process starts it also passes through the system startup code, but
48 ; it does not initialization the kernel debugger.
49 ;
50 ; Arguments:
51 ;
52 ; LoaderBlock (rcx) - Supplies a pointer to the loader block.
53 ;
54 ; Implicit Arguments:
55 ;
56 ; When the system starts up the loader has done some initialization. In
57 ; particular all structures have at least been zeroed and the GDT and
58 ; TSS have been completely initialized.
59 ;
60 ; The loader block has been reformatted by the loader into a 64-bit loader
61 ; block and all pertinent fields have been filled in.
62 ;
63 ; The address of the PRCB is passed in the loader block (only for processors
64 ; other than zero).
65 ;
66 ; The address of the idle thread and idle process are passed in the loader
67 ; block (only for processors other than zero).
68 ;
69 ; The GDT and IDT address and limits are contained in the gdtr and idtr
70 ; registers.
71 ;
72 ; The address of the TSS must be extraced from the appropriate GDT entry
73 ; and stored in the PCR.
74 ;
75 ; The stack register (RSP) is loaded with the idle thread stack and the
76 ; kernel stack field of the loader block contains the address of the DPC
77 ; stack.
78 ;
79 ; Return Value:
80 ;
81 ; None - function does not return.
82 ;
83 ;--
