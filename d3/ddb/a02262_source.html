542 //
543 // This section contains extended enable flags whcih are private.
544 //
545 // Each PerfMacros Hook Contains a GlobalMask and a Hook Id.
546 // The Global Mask is Used For Grouping Hooks by logical type
547 // - I/O related Hooks are Grouped together under
548 // PERF_FILE_IO or PERF_DISK_IO
549 // - Loader related Hooks are grouped together
550 // under PERF_LOADER,
551 // - etc
552 // The data for a particular hook will only be logged
553 // if the Global Mask of the particular Hook is set.
554 //
555 // WHEN YOU ADD NEW GROUPS, UPDATE THE NAME TABLE in perfgroups.c:
556 // PerfGroupNames Note: If you modify numeric value of a group, update
557 // PerfKnownFlags table
558 //
559 // we have a set of 8 global masks available. the highest 3 bits in
560 // PERF_MASK_INDEX region determine to which set a particular
561 // global group belongs. if PERF_MASK_INDEX is 0xe0000000
562 // all of the following can be unique groups that can be
563 // turned on or of individually and used when logging data:
564 //
565 // #define PERF_GROUP1 0x00400000 in the 0th set
566 // #define PERF_GROUP2 0x20400000 in the 1st set
567 // #define PERF_GROUP3 0x40400000 in the 2nd set
568 // ...
569 // #define PERF_GROUP2 0xe0400000 in the 7th set
570 //
571 // See ntperf.h for the manipulation of flags
572 //
573 //
574 // Currently, no GlobalMask change is supported.
575 //
576 // Merging logging with WMI, we will use the first global mask for flags used
577 // by both PERF and WMI
