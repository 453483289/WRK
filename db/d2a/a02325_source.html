40 Routine Description:
41 
42  This service builds descriptors or MDLs for the supplied buffer(s) and
43  passes the untyped data to the device driver associated with the file
44  handle. It is up to the driver to check the input data and function
45  IoControlCode for validity, as well as to make the appropriate access
46  checks.
47 
48 Arguments:
49 
50  FileHandle - Supplies a handle to the file on which the service is being
51  performed.
52 
53  Event - Supplies an optional event to be set to the Signaled state when
54  the service is complete.
55 
56  ApcRoutine - Supplies an optional APC routine to be executed when the
57  service is complete.
58 
59  ApcContext - Supplies a context parameter to be passed to the ApcRoutine,
60  if an ApcRoutine was specified.
61 
62  IoStatusBlock - Address of the caller's I/O status block.
63 
64  IoControlCode - Subfunction code to determine exactly what operation is
65  being performed.
66 
67  InputBuffer - Optionally supplies an input buffer to be passed to the
68  device driver. Whether or not the buffer is actually optional is
69  dependent on the IoControlCode.
70 
71  InputBufferLength - Length of the InputBuffer in bytes.
72 
73  OutputBuffer - Optionally supplies an output buffer to receive information
74  from the device driver. Whether or not the buffer is actually optional
75  is dependent on the IoControlCode.
76 
77  OutputBufferLength - Length of the OutputBuffer in bytes.
78 
79 Return Value:
80 
81  The status returned is success if the control operation was properly
82  queued to the I/O system. Once the operation completes, the status
83  can be determined by examining the Status field of the I/O status block.
