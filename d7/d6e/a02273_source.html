127 //
128 // Caveat Programmer:
129 //
130 // The pool header must be QWORD (8 byte) aligned in size. If it
131 // is not, the pool allocation code will trash the allocated
132 // buffer.
133 //
134 //
135 //
136 // The layout of the pool header is:
137 //
138 // 31 23 16 15 7 0
139 // +----------------------------------------------------------+
140 // | Current Size | PoolType+1 | Pool Index |Previous Size |
141 // +----------------------------------------------------------+
142 // | ProcessBilled (NULL if not allocated with quota) |
143 // +----------------------------------------------------------+
144 // | Zero or more longwords of pad such that the pool header |
145 // | is on a cache line boundary and the pool body is also |
146 // | on a cache line boundary. |
147 // +----------------------------------------------------------+
148 //
149 // PoolBody:
150 //
151 // +----------------------------------------------------------+
152 // | Used by allocator, or when free FLINK into sized list |
153 // +----------------------------------------------------------+
154 // | Used by allocator, or when free BLINK into sized list |
155 // +----------------------------------------------------------+
156 // ... rest of pool block...
157 //
158 //
159 // N.B. The size fields of the pool header are expressed in units of the
160 // smallest pool block size.
161 //
162 // N.B. AMD64 has a 16 byte allocation size and 4096 byte page size. Therefore,
163 // byte fields in the pool header are adequate.
164 //
165
