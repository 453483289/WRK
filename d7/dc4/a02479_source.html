95 /*++
96 
97 Routine Description:
98 
99  A client process can connect to a server process by name using the
100  NtConnectPort service.
101 
102  The PortName parameter specifies the name of the server port to
103  connect to. It must correspond to an object name specified on a
104  call to NtCreatePort. The service sends a connection request to the
105  server thread that is listening for them with the NtListenPort
106  service. The client thread then blocks until a server thread
107  receives the connection request and responds with a call to the
108  NtCompleteConnectPort service. The server thread receives the ID of
109  the client thread, along with any information passed via the
110  ConnectionInformation parameter. The server thread then decides to
111  either accept or reject the connection request.
112 
113  The server communicates the acceptance or rejection with the
114  NtCompleteConnectPort service. The server can pass back data to the
115  client about the acceptance or rejection via the
116  ConnectionInformation data block.
117 
118  If the server accepts the connection request, then the client
119  receives a communication port object in the location pointed to by
120  the PortHandle parameter. This object handle has no name associated
121  with it and is private to the client process (i.e. it cannot be
122  inherited by a child process). The client uses the handle to send
123  and receive messages to/from the server process using the
124  NtRequestWaitReplyPort service.
125 
126  If the ClientView parameter was specified, then the section handle
127  is examined. If it is a valid section handle, then the portion of
128  the section described by the SectionOffset and ViewSize fields will
129  be mapped into both the client and server process' address spaces.
130  The address in client address space will be returned in the ViewBase
131  field. The address in the server address space will be returned in
132  the ViewRemoteBase field. The actual offset and size used to map
133  the section will be returned in the SectionOffset and ViewSize
134  fields.
135 
136  If the server rejects the connection request, then no communication
137  port object handle is returned, and the return status indicates an
138  error occurred. The server may optionally return information in the
139  ConnectionInformation data block giving the reason the connection
140  requests was rejected.
141 
142  If the PortName does not exist, or the client process does not have
143  sufficient access rights then the returned status will indicate that
144  the port was not found.
145 
146 Arguments:
147 
148  PortHandle - A pointer to a variable that will receive the client
149  communication port object handle value.
150 
151  PortName - A pointer to a port name string. The form of the name
152  is [\name...\name]\port_name.
153 
154  SecurityQos - A pointer to security quality of service information
155  to be applied to the server on the client's behalf.
156 
157  ClientView - An optional pointer to a structure that specifies the
158  section that all client threads will use to send messages to the
159  server.
160 
161  ClientView Structure
162 
163  ULONG Length - Specifies the size of this data structure in
164  bytes.
165 
166  HANDLE SectionHandle - Specifies an open handle to a section
167  object.
168 
169  ULONG SectionOffset - Specifies a field that will receive the
170  actual offset, in bytes, from the start of the section. The
171  initial value of this parameter specifies the byte offset
172  within the section that the client's view is based. The
173  value is rounded down to the next host page size boundary.
174 
175  ULONG ViewSize - Specifies a field that will receive the
176  actual size, in bytes, of the view. If the value of this
177  parameter is zero, then the client's view of the section
178  will be mapped starting at the specified section offset and
179  continuing to the end of the section. Otherwise, the
180  initial value of this parameter specifies the size, in
181  bytes, of the client's view and is rounded up to the next
182  host page size boundary.
183 
184  PVOID ViewBase - Specifies a field that will receive the base
185  address of the section in the client's address space.
186 
187  PVOID ViewRemoteBase - Specifies a field that will receive
188  the base address of the client's section in the server's
189  address space. Used to generate pointers that are
190  meaningful to the server.
191 
192  RequiredServerSid - Optionally specifies the SID that we expect the
193  server side of the port to possess. If not specified then we'll
194  connect to any server SID.
195 
196  ServerView - An optional pointer to a structure that will receive
197  information about the server process' view in the client's
198  address space. The client process can use this information
199  to validate pointers it receives from the server process.
200 
201  ServerView Structure
202 
203  ULONG Length - Specifies the size of this data structure in
204  bytes.
205 
206  PVOID ViewBase - Specifies a field that will receive the base
207  address of the server's section in the client's address
208  space.
209 
210  ULONG ViewSize - Specifies a field that will receive the
211  size, in bytes, of the server's view in the client's address
212  space. If this field is zero, then server has no view in
213  the client's address space.
214 
215  MaxMessageLength - An optional pointer to a variable that will
216  receive maximum length of messages that can be sent to the
217  server. The value of this parameter will not exceed
218  MAX_PORTMSG_LENGTH bytes.
219 
220  ConnectionInformation - An optional pointer to uninterpreted data.
221  This data is intended for clients to pass package, version and
222  protocol identification information to the server to allow the
223  server to determine if it can satisfy the client before
224  accepting the connection. Upon return to the client, the
225  ConnectionInformation data block contains any information passed
226  back from the server by its call to the NtCompleteConnectPort
227  service. The output data overwrites the input data.
228 
229  ConnectionInformationLength - Pointer to the length of the
230  ConnectionInformation data block. The output value is the
231  length of the data stored in the ConnectionInformation data
232  block by the server's call to the NtCompleteConnectPort
233  service. This parameter is OPTIONAL only if the
234  ConnectionInformation parameter is null, otherwise it is
235  required.
236 
237 Return Value:
238 
239  NTSTATUS - An appropriate status value.
240 
241 --*/
