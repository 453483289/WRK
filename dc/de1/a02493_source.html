63 /*++
64 
65 Routine Description:
66 
67  This function creates a region of pages within the virtual address
68  space of a subject process.
69 
70 Arguments:
71 
72  ProcessHandle - Supplies an open handle to a process object.
73 
74  BaseAddress - Supplies a pointer to a variable that will receive
75  the base address of the allocated region of pages.
76  If the initial value of this argument is not null,
77  then the region will be allocated starting at the
78  specified virtual address rounded down to the next
79  host page size address boundary. If the initial
80  value of this argument is null, then the operating
81  system will determine where to allocate the region.
82 
83  ZeroBits - Supplies the number of high order address bits that
84  must be zero in the base address of the section view. The
85  value of this argument must be less than or equal to the
86  maximum number of zero bits and is only used when memory
87  management determines where to allocate the view (i.e. when
88  BaseAddress is null).
89 
90  If ZeroBits is zero, then no zero bit constraints are applied.
91 
92  If ZeroBits is greater than 0 and less than 32, then it is
93  the number of leading zero bits from bit 31. Bits 63:32 are
94  also required to be zero. This retains compatibility
95  with 32-bit systems.
96 
97  If ZeroBits is greater than 32, then it is considered as
98  a mask and then number of leading zero are counted out
99  in the mask. This then becomes the zero bits argument.
100 
101  RegionSize - Supplies a pointer to a variable that will receive
102  the actual size in bytes of the allocated region
103  of pages. The initial value of this argument
104  specifies the size in bytes of the region and is
105  rounded up to the next host page size boundary.
106 
107  AllocationType - Supplies a set of flags that describe the type
108  of allocation that is to be performed for the
109  specified region of pages. Flags are:
110 
111  MEM_COMMIT - The specified region of pages is to be committed.
112 
113  MEM_RESERVE - The specified region of pages is to be reserved.
114 
115  MEM_TOP_DOWN - The specified region should be created at the
116  highest virtual address possible based on ZeroBits.
117 
118  MEM_RESET - Reset the state of the specified region so
119  that if the pages are in a paging file, they
120  are discarded and if referenced later, pages of zeroes
121  are materialized.
122 
123  If the pages are in memory and modified, they are marked
124  as not modified so they will not be written out to
125  the paging file. The contents are NOT zeroed.
126 
127  The Protect argument is ignored, but a valid protection
128  must be specified.
129 
130  MEM_PHYSICAL - The specified region of pages will map physical memory
131  directly via the AWE APIs.
132 
133  MEM_LARGE_PAGES - The specified region of pages will be allocated from
134  physically contiguous (non-paged) pages and be mapped
135  with a large TB entry.
136 
137  MEM_WRITE_WATCH - The specified private region is to be used for
138  write-watch purposes.
139 
140  Protect - Supplies the protection desired for the committed region of pages.
141 
142  PAGE_NOACCESS - No access to the committed region
143  of pages is allowed. An attempt to read,
144  write, or execute the committed region
145  results in an access violation.
146 
147  PAGE_EXECUTE - Execute access to the committed
148  region of pages is allowed. An attempt to
149  read or write the committed region results in
150  an access violation.
151 
152  PAGE_READONLY - Read only and execute access to the
153  committed region of pages is allowed. An
154  attempt to write the committed region results
155  in an access violation.
156 
157  PAGE_READWRITE - Read, write, and execute access to
158  the committed region of pages is allowed. If
159  write access to the underlying section is
160  allowed, then a single copy of the pages are
161  shared. Otherwise the pages are shared read
162  only/copy on write.
163 
164  PAGE_NOCACHE - The region of pages should be allocated
165  as non-cachable.
166 
167 Return Value:
168 
169  Various NTSTATUS codes.
170 
171 --*/
